Introduction¶

Dataset Description¶

TMDB databse is provided by Kaggle for those who are intrested in data analysis/science and machine learning. This database contains over 10,000 records about some movies, including infomration about their genres, casts, audience ratings, and revenues.

The database is composed of one single table that has 21 columns listed in the following data dictionary.

Column Name Data Type Description Allows Null
id int64 Record id. Used as a primary key No
imdb_id str IMDB record id Yes
popularity float64 Movie popularity rate No
budget float64 Movie budget No
revenue float64 Movie revenue No
original_title str Movie title No
cast str Movie cast Yes
homepage str Movie website Yes
director str Movie director Yes
tagline str Movie tagline Yes
keywords str Keywords for search engines Yes
overview str Overview about the plot Yes
runtime int64 Movie length No
genres str Movie genres Yes
production_companies str Movie production companies Yes
release_date str Movie release date No
vote_count int64 Audience vote count No
vote_average float64 Audience vote average (out of 10) No
release_year int64 Movie release year No
bugject_adj float64 Movie budget in terms of 2010 US Dollars, accounting for inflation No
revenue_adj float64 Movie revenue in terms of 2010 US Dollars, accounting for inflation No

Question(s) for Analysis¶

This report aims at analyzing the provided dataset in order to get better insights about movies. To be specific, we will try to answer the following questions using basic data analysis techniques, as well as Python Pandas, Numpy and Matplotlib libraries:

  1. Which genres are most popular from year to year?
  2. For how many years each of the most popular genres and least popular genres appeared in the ranking lists?
  3. What kinds of properties are associated with movies that have high revenues?

In order to achieve this goal, the provided dataset first needs to be cleaned. A good understanding of the provided dataset and the issues that come with it, as well as fixing these issues, is essential for a successful data analysis. In the subsequent sections, we will try to accomplish this, first by cleaning and transofrming our dataset, or what is commonly known as data wrangeling, and then we will explore the presented variables and the relationships between them.

def analyze(col_name, col_alias):
    """Draws a bar chart depicting the corelation between the 4 revenue levels and the targeted column
    
    Parameters
    ----------
    col_name: str
        The column to be analyzed
    col_alias: str
        An alias to used to replace the column name with a friendly name in the generated graph
    Returns
    -------
        None
    """
    
    df_low = df.query("revenue_level == 'low'")
    df_medium = df.query("revenue_level == 'medium'")
    df_high = df.query("revenue_level == 'high'")
    df_vhigh = df.query("revenue_level == 'very high'")
    
    mean_low = df_low[col_name].mean()
    mean_medium = df_medium[col_name].mean()
    mean_high = df_high[col_name].mean()
    mean_vhigh = df_vhigh[col_name].mean()

    #Creates 4 items
    locations = np.arange(1, 5, 1)
    heights = [mean_vhigh, mean_high, mean_medium, mean_low]
    labels = df["revenue_level"].dropna().unique().tolist()
    labels = [label.capitalize() for label in labels]         #Capitalize labels
    
    plt.figure(figsize=(7, 7))
    plt.bar(locations, heights, tick_label=labels)
    plt.xlabel("Revenue Level")
    plt.ylabel(col_alias)
    plt.title(f"{col_alias} per Revenue Levels")
    plt.show();
    
    # Group by the revenue level and get column statistics (means) 
    df_stats = df.groupby("revenue_level", as_index=False)[col_name].mean()
    print(df_stats.to_string(index=False))

Conclusions¶

In this project, we managed to do some analysis for the TMDB database. We tried to answer three questions, and the findings are interesting.

First, we managed to create a list of the most popular genres, as well as a list of the least popular genres. The analysis shows that genre popularity vary in both lists from year to year.
